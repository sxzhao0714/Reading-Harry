# 任务：章节高亮总结

## 输入
- 目标章节文件路径（用户运行时指定）
- 规则文件：`00-Meta/Color-Convention.md`

## 执行步骤

### 1. 读取与解析
1.1 读取目标章节文件全文。
1.2 仅处理"## 原文 Original"和"## AI 总结 Summary"之间的内容；"## 我的笔记 My Notes"区永不读取、永不写入。
1.3 用正则抽取所有高亮：
    `<mark class="eme-highlight eme-h-(yellow|blue|green|pink)" data-id="[^"]+">([^<]+)</mark>`
    捕获颜色与文字，并保留每条高亮所在的句子作为上下文。
1.4 按颜色分组统计数量，作为完成报告的一部分。

### 2. 保留用户已编辑条目
2.1 在覆盖 AI-SUMMARY 区之前，先扫描旧内容中所有以 `[edited]` 结尾的条目，原样保留。
2.2 新生成的内容与保留条目合并：保留条目优先，避免重复生成同一词条。

### 3. 调用 LLM 生成总结
将以下提示词发送给 LLM，连同所有高亮（按颜色分组、附上下文）一起：

---
````
 你是一位专为 CET-4 水平、目标 CET-6/雅思的中国英语学习者服务的语言学习助手。读者正在读《哈利·波特》英文原版。请按下方规则，把我提供的所有高亮整理为四个小节。

 **总规则：**
 - 不要编撰原文以外的剧情。
 - 不要输出寒暄或开场白。
 - 严格按指定格式输出，便于程序化处理。
 - 同一词的不同形态（plural / past tense / -ing）合并为一条，例句可保留多条。
 - 同一颜色内重复出现的相同条目去重。

 **黄色 = 生词。** 每条格式：
 ```
 - **词** /音标/ 词性. 中文释义
   - 例: "原文中的最小完整短语或子句"
   - 笔记：（仅当有明显特殊用法、词源、或罗琳偏好时才写一句；否则不写此行）
 ```
 释义不要附英文解释。释义贴合 CET-6/雅思水平，不要过度简化。

 **蓝色 = 精彩表达。** 每条格式：
 ```
 - **原表达**
   - 含义：中文解释
   - 用法：（关键搭配 / 词性 / 语境）
   - 模仿：用学习者能套用的英文造句一例
 ```

 **绿色 = 句型 / 语法 / 卡住的短语。** 每条格式：
 ```
 - **原表达**（标签：习语 / 短语动词 / 句型 / 语法点 / 方言 / 倒装 等）
   - 含义：中文解释
   - 结构 / 用法：拆解或说明关键点
   - 例：英文例句一例
 ```
 若是 Hagrid 等角色的方言（如 ter, swarmin', an'），合并为一条"方言点"，列出本章出现的所有形式与对应标准拼写，不必逐个造句。

 **粉色 = 用户标注。** 每条格式：
 ```
 - > 原文片段
   - 译：中文翻译（专有名词补一句简短背景，如"英格兰西南城市"）
 ```

 **输出结构：**
 严格按如下四级标题输出，标题文字一字不差：
 ```
 ### 生词（Yellow）

 （条目）

 ### 精彩表达（Blue）

 （条目）

 ### 句型 / 语法（Green）

 （条目）

 ### 标注（Pink）

 （条目）
 ```
 若某颜色本章无高亮，该小节下写一行"（本章无）"。
````

---

### 4. 生成"本章人物动态"小节
4.1 单独再调用一次 LLM（或在同一次输出中），基于章节原文（去除 mark 标签后的纯文本）生成一段简短的"本章人物动态"，结构如下：

```
### 本章人物动态

出场角色（白名单内）：…
提及但未登场（白名单内）：…
提及（白名单外）：…

剧情要点：一段不超过 80 字的中文概要。
```

判定"白名单内"时读取 `00-Meta/Character-Whitelist.md` 的 folder_name 列。
"白名单外"角色仅列名，不在 02-Characters/ 下建文件。

### 5. 写回文件
5.1 将上述五个小节（生词 / 精彩表达 / 句型 / 标注 / 本章人物动态）合并，整体替换 `<!-- AI-SUMMARY-START -->` 与 `<!-- AI-SUMMARY-END -->` 之间的内容。
5.2 注释行 `<!-- 此区块由 TRAE 自动生成。修改后请在该条目末尾加 [edited]，AI 重跑时会跳过 -->` 必须保留在锚点正下方。
5.3 步骤 2 中保留的 `[edited]` 条目，按所属小节归位。
5.4 更新章节 frontmatter：
    - `last_analyzed`: 当前日期，格式 `YYYY-MM-DD`
    - `status`: 若当前为 `done`，改为 `analyzed`；若已是 `analyzed`，保持。
    - `characters_in_chapter`: 写入"本章人物动态"中"出场角色（白名单内）"列表的 folder_name（数组）。

### 6. 完成报告
输出一份纯文本报告，包含：
- 处理的章节文件名
- 各颜色高亮原始数量、去重后条目数
- 保留的 `[edited]` 条目数
- 识别出的白名单角色（folder_name 列表）
- 提及的白名单外角色名

## 严格禁止
- 不得修改"## 原文 Original"区任何字符。
- 不得修改"## 我的笔记 My Notes"区任何字符。
- 不得修改 AI-SUMMARY 区锚点之外的任何内容。
- 不得调用人物抽取流程（那是另一个提示词的职责）。
